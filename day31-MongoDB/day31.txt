1.关系数据库
    关系数据库：使用二维表表示数据，数据联系
    DBMS：数据库管理系统，一套数据库管理的软件

    表：由行、列组成，用来表示一定意义的数据
    字段：标的一列，表示实体的一个属性
    键：可以区分实体的属性（属性组合）
    主键：从键中选一个作为主键
    索引：提高查询效率的技术
         通过避免全表扫描提高查询效率
    缺点：
        性能：在高并发、大数据量的情况下，性能容易成为瓶颈
              为了维护数据的一致性，需要在写数据的时候对数据加锁
              连表查询容易导致性能下降
        扩展性：无法通过横向扩展来扩展性能、负载能力
        处理非结构化数据不方便

2.非关系型数据库
    1.优点：
        高并发、读写能力强
        易扩展
        弱化了数据结构，降低数据一致性要求
    2.缺点：
        技术成熟度不如关系型数据库
        通用性较差，不如SQL语言有统一标准
        操作冗长，容易混乱
        有些不支持jion/事务，一致性不如关系型
    3.使用非关系型数据库的情况
        对数据一致性要求较低
        数据库需要处理海量并发
        需要快速处理数据库

3.MongoDB的特点
    开发语言：C++
    支持丰富增删改查，最像关系型数据库的非关系型数据库
    技术相对成熟，支持丰富的存储类型
    支持众多的编程语言接口
    使用方便，便于扩展、部署

4.MongoDB 安装
    1.Ubuntu：sudo apt-get install mongodb
        默认安装位置： /var/lib/mongodb
        配置文件： /etc/mongodb.conf
        命令集： /usr/bin
    2.手工安装
        设置环境变量：将/opt/mongo/bin加入到环境变量： PATH=$PATH:/opt/mongodb/bin          export PATH
    3.启动：
        /opt/mongo/bin/mongod --dbpath=/user/mongo_data
    4.查看服务
        看端口：默认为27017端口     netstat -an | grep 27017
    5.基本操作
        1.mongo --dbpath 目录
            功能：设置mongodb数据存储路径
            e.g.mongod --dbpath /home/tarena/mongo/
        2.mongod --port 端口号
            功能：修改服务监听的端口，默认27017
        3.启动mongo操作界面（mongo shell）
            mongo   登录到本机
            mongo  127.0.0.1:27017
                   连接到127.0.0.1:27017服务器
            mongo  192.168.1.10/test -u xxx -p xxx
                   使用指定的用户名密码登录远程数据库
                   test表示库名称
    6.mongo 数据结构
        键值对 --> 文档 --> 集合 --> 数据库 
        关系数据库概念          mongodb概念
        ----------------------------------
            数据库              数据库
            表                  集合（collection）
            行                  文档（document, 一笔数据）
            列                  域（field）
            索引                索引 
            主键                主键 
    7.库操作
        1.创建库：use databaseName
            说明：表示进入某个库，当库不存在时自动创建，在实际写入数据时创建，而不是马上创建
            例： use test --->  创建test库
        2.查看所有库：show dbs 
            系统中会有几个自动创建的库
                admin:该库中的成员拥有最高权限
                local:存储跟本地服务器相关的数据，这部分数据不会被赋值到其他服务器
        3.查看当前所在库：db
        4.删除库：db.dropDatabase()
            删除之前首先要进入要删除的库
        5.库的命名规则
            使用utf-8字符
            不能有空格，点，/，\，'\0'(空字符)
            长度不能唱过64字节
            不能和系统数据库重名
        6.备份和恢复
            备份：mongodump -h 主机 -d 库名 -o 目录
            例：mongodump -h localhost -d test -o /tmp/
            恢复：mongorestore -h 主机：端口 -d 库名 路径
        7.数据库监控：
            mongostat:
                insert query update delete:每秒执行增删改查次数
                command：每秒运行的命令次数
                flushes：每秒清理缓存的次数
                vsize：虚拟内存的使用量
                res：物理内存使用量
            mongotop:
                ns:数据集合
                total:总时长
                read:读时长
                write:写时长
        8.集合操作
            1.集合：相当于关系数据库中的表，包含很多文档，集合没有特定的结构，当有数据插入时自动创建
            2.集合的使用
                同一类的数据放入一个集合，尽量保持每条数据结构的一致性
            3.集合的操作命令
                创建：
                    格式：db.createCollection(集合名称)
                删除集合：
                    格式：db.collectionName.drop()
                    例：db.acct.drop()
                查询集合：
                    格式：db.collectionName.find().pretty()
                    例：db.acct.find().pretty()
                重命名集合：
                    格式：db.collectionName.renameCollection(集合名)
                    例： db.student.renameCollection('stun')  
        9.文档操作
            1.文档：文档是构成集合的基本单元，相当于关系数据库的行（记录），由一系列键值对组成
                文档的特点
                    1.文档中的键值对是有序的
                    2.mongodb文档中的键不能重复
                    3.名称区分大小写
                    4.键需要使用utf-8字符
                存储：
                    存储为BSON(Binary JSON) ----> 二进制存储
                文档插入：
                    格式：db.collectionName.insert([{....},{...},...])
                          db.collectionName.save({....})   有数据修改，没数据插入
                ObjectId:系统为每一个文档生成的主键
                    长度24字符
                    8位：文档创建时间
                    6位：机器ID
                    4位：进程ID
                    6位：流水号
            2，MongoDb数据类型
                Double              1       浮点型
                String              2       字符串（utf-8）
                Object              3       内嵌文档
                Array               4       数组
                Binary Data         5       二进制（图片，视频）
                ObjectId            7       系统生成的ObjectId
                Boolean             8       布尔（true,false）
                Date                9       日期时间
                NULL                10      空值
                Integer             16/18   整形（32b/64b整型）
                Timestamp           17      时间戳
